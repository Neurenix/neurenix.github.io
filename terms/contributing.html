Contribution Terms

Last Update: 03/08/2025

Thank you for your interest in contributing to Neurenix! This is an open-source project, and as part of the community, your contributions are crucial to its success. To ensure a healthy and efficient collaboration environment, please follow these guidelines when contributing to the project.

License:

Neurenix is licensed under the Apache 2.0 License. By contributing code, you grant Neurenix an unrestricted, irrevocable, and royalty-free license to use, modify, distribute, and sublicense your contributions as described by the license.

What You Can Do:

- Report bugs

- Suggest improvements or new features

- Contribute code

- Contribute documentation

What You Cannot Do:

- Violate copyrights

- Submit code without proper authorization or under a license incompatible with the Apache 2.0 License

Reporting Bugs:

If you encounter a bug, please report it in detail via GitHub Issues. For security issues, do not use GitHub Issues or Discussions. Instead, please contact our security team directly at security@milesonerd.8shield.net.

Suggesting Improvements:

If you have an idea for an improvement or a new feature, share it in the GitHub Discussions. If the suggestion is relevant and useful, it may be incorporated into the future of Neurenix!

Code of Conduct:

Neurenix follows a Code of Conduct based on the Contributor Covenant. By contributing, you agree to adhere to this code, ensuring that all contributors are treated with respect and kindness. In short:

- Be respectful: Treat everyone with dignity and respect.

- Be inclusive: Embrace diversity and ensure the environment is welcoming to all.

- Be collaborative: Work constructively and provide positive feedback.

Violations of these principles may result in being blocked on social media or expelled from the community.
